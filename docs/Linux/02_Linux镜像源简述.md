## 一、镜像源介绍

本质是 软件包仓库的 “副本服务器”，核心作用是存储 Linux 系统所需的所有软件包（如工具、库文件、应用程序）及依赖关系，让你通过 yum/apt/pacman 等命令一键下载、安装、更新软件，不用手动找安装包和解决依赖。

## 二、镜像源核心作用

### 2.1 提供软件包下载

系统自带的软件（如 vim、wget）、第三方工具（如 nginx、MySQL），都能从镜像源直接获取，无需官网逐个下载.

### 2.2 解决依赖关系

安装软件时往往需要依赖其他库文件（如安装 MySQL 需 libaio），镜像源会自动识别并下载所有依赖，避免 “缺依赖导致安装失败”。

### 2.3 软件更新升级

通过 yum update/apt upgrade 命令，能从镜像源获取软件的最新版本，修复漏洞、新增功能。

### 2.4 加速访问

官方镜像源（如 CentOS 官方源）通常在国外，国内镜像源（阿里云、清华源）是其 “副本”，部署在国内机房，下载速度更快、更稳定，避免国外链路超时失效

## 三、镜像源的分类

### 3.1 官方镜像源

发行版官方提供，如 CentOS 的 `mirror.centos.org`、Ubuntu 的 `archive.ubuntu.com`。这类镜像源的软件包最全面、最权威，但由于服务器可能位于国外，在国内访问时速度可能较慢。

### 3.2 第三方镜像源

高校或企业镜像站提供，如清华大学的 `mirrors.tuna.tsinghua.edu.cn`、阿里云的 `mirrors.aliyun.com`。这些镜像源对官方软件仓库进行了同步，在国内访问速度较快，并且会及时更新软件包。

## 四、更换镜像源原因

在国内由于各种原因下载或更新国外的软件网速特别慢甚至连不上，因此需要更换为国内的镜像源，从而加速软件包的下载和更新过程，提高系统的稳定性和效率。

## 五、常见镜像源

### 5.1 阿里云镜像源

阿里云提供，速度快且稳定，对各种 Linux 发行版支持良好。

### 5.2 清华大学镜像源

清华大学开源软件镜像站，提供了丰富的开源软件镜像，包括常见的 Linux 发行版。

### 5.3 中科大镜像源：

中国科学技术大学的开源软件镜像，更新及时，访问速度也不错。

## 六、示例

更换镜像源示例（以 CentOS 7 更换阿里云镜像源为)

### 6.1 备份原镜像源配置文件

```
sudo mv /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.backup
```


### 6.2 下载阿里云镜像源配置文件

```
sudo wget -O /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo
```

### 6.3 脚本更换镜像源

也可以通过脚本换源(由社区开发者SuperManito 制作和维护。它能智能识别系统版本、测试镜像站速度，并自动备份和替换源)
```
bash <(curl -sSL https://linuxmirrors.cn/main.sh)
```

### 6.4 清理并生成缓存
```
sudo yum clean all sudo yum makecache
```
