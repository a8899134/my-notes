## 二、目录概念
目录的概念不是文件集合。目录和文件一样，目录也是文件。目录是找到文件的“踏板”。目录的本质是路径映射。
## 三、Linux 目录结构
根目录
所有的文件都需要从根开始。Linux 只要一个根。就像一颗树。Linux 的根与内核直接关联。Linux 的根用 / 表示。
一级子目录
1. /bin：存放二进制可执行文件（binary），包含系统启动和日常操作所需的基本命令（如 ls、cp 等），所有用户均可使用。
2. /boot：包含启动 Linux 系统所需的文件（如内核、启动加载程序等），是系统启动的关键目录。
3. /dev：设备文件目录（devices），Linux 中所有硬件设备都以文件形式在此表示（如硬盘 /dev/sda、键盘 /dev/input/event 0 等）。
4. /etc：存放系统配置文件（etcetera），包括用户账号配置、服务配置等（如 /etc/passwd、/etc/ssh/sshd_config 等）。
5. /home：普通用户的主目录集合，每个用户在此有一个以用户名命名的子目录（如 /home/user 1），用于存储用户个人文件和设置。
6. /lib：存放共享库文件（libraries），这些库是二进制程序运行所需的依赖文件，类似 Windows 的 DLL 文件。
7. /lib 64：针对 64 位系统的共享库目录，与 /lib 功能类似，用于存放 64 位程序的依赖库。
8. /media：可移动媒体设备的挂载点（media），如 USB 闪存盘、光盘等通常会自动挂载到此处的子目录。
9. /mnt：临时挂载目录（mount），用户可手动将外部存储设备或文件系统挂载到这里。
10. /opt：可选应用程序目录（optional），用于安装第三方软件（如商业软件），通常由软件自行创建子目录。
11. /proc：伪文件系统目录（processes），包含当前运行进程和系统内核的实时信息（如 /proc/cpuinfo 显示 CPU 信息），内容存在于内存中。
12. /root：超级用户（root 用户）的主目录，与普通用户的 /home 目录功能类似，但位置独立。
13. /run：存放系统运行时的临时数据（run），如进程 PID 文件、临时挂载信息等，系统重启后会重新创建。
14. /sbin：系统二进制文件（system binary），包含用于系统管理的命令（如 reboot、ifconfig 等），通常需要 root 权限才能执行。
15. /srv：服务数据目录（service），存放系统提供服务时所需的数据（如网页服务器的网站文件可放在 /srv/www）。
16. /sys：伪文件系统目录（system），用于与内核交互，提供硬件设备和系统内核的相关信息（如 /sys/class/net 显示网络接口）。
17. /tmp：临时文件目录（temporary），所有用户可在此创建临时文件，系统重启后通常会清空。
18. /usr：用户共享资源目录（user），包含大部分用户工具和应用程序，类似 Windows 的 Program Files，其子目录如 /usr/bin（用户命令）、/usr/lib（用户库）等。
19. /var：可变数据目录（variable），存放经常变化的数据（如日志文件 /var/log、邮件 /var/mail、缓存 /var/cache 等）。
