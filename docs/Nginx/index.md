# Nginx 笔记

Nginx（读作"engine x"）是一款开源、高性能的 HTTP 服务器和反向代理服务器，同时也可用作邮件代理服务器（IMAP/POP3/SMTP）。由俄罗斯程序员 Igor Sysoev 为解决 **C10K 问题**（单台服务器同时处理一万个并发连接）而设计，于 2004 年首次公开发布。如今，Nginx 已成为全球最流行的 Web 服务器之一，百度、京东等众多大型网站均采用 Nginx 作为核心 Web 组件。

Nginx 官方文档网址：https://nginx.org/en/docs/

Nginx 官方软件包仓库网址：https://nginx.org/packages/
## 为什么选择 Nginx

- **高性能高并发**：采用事件驱动的异步非阻塞架构，单机可处理数万并发连接，有报告表明能支持高达 50,000 个并发连接数
- **内存占用低**：轻量级设计，资源消耗极小，同类型网页服务器中表现突出
- **高可靠性**：支持热部署，几乎可以做到 7×24 小时不间断运行，即使运行几个月也不需要重新启动
- **开源免费**：采用 2-clause BSD 许可证，允许商业使用且无强制开源要求
- **生态丰富**：模块化设计，功能可灵活扩展，社区活跃，文档完善

## 核心功能

| 功能 | 说明 |
| :--- | :--- |
| **静态资源服务器** | 托管 HTML、CSS、JS、图片等静态文件，处理速度极快 |
| **反向代理** | 接收客户端请求并转发给后端应用服务器，隐藏真实服务架构 |
| **负载均衡** | 支持轮询、加权轮询、IP 哈希、最少连接数等多种策略，智能分配请求压力 |
| **动静分离** | 动态请求与静态请求分离处理，提升整体性能 |
| **SSL/TLS 终结** | 集中处理 HTTPS 加密解密，减轻后端服务器负担 |
| **API 网关** | 微服务架构中统一处理请求路由、认证鉴权、限流熔断等 |
| **邮件代理** | 支持 IMAP/POP3/SMTP 协议代理 |

## 架构原理

Nginx 采用 **Master-Worker 多进程架构**：

- **Master 进程**：不处理网络请求，主要负责加载配置、启动工作进程及平滑升级
- **Worker 进程**：负责处理网络请求与响应，采用事件驱动模型，各进程间通过共享内存 + 原子操作实现无锁通信

其高性能的核心在于 **事件驱动的异步非阻塞 I/O 模型**，底层使用操作系统提供的 I/O 多路复用机制（如 Linux 的 `epoll`、BSD 的 `kqueue`）。与传统服务器为每个请求创建独立线程或进程不同，Nginx 的每个工作进程通过事件循环处理多个连接请求，避免了线程切换开销。

## 典型应用场景

- **个人网站或公司官网**：作为 Web 服务器直接托管站点内容
- **外部网关代理**：作为统一入口，屏蔽内部服务细节
- **微服务流量治理**：在微服务架构中承担 API 网关角色
- **高并发业务支撑**：电商大促、秒杀抢购等场景下的流量洪峰应对
- **CDN 节点**：作为边缘节点提供高速内容分发

> 📌 **总结**：Nginx 凭借其事件驱动的非阻塞架构、低内存占用和高可靠性，已成为现代互联网基础设施中不可或缺的核心组件。无论是作为静态服务器、反向代理还是负载均衡器，Nginx 都能以极低的资源消耗支撑海量用户并发访问，是每一位运维和开发人员必须掌握的核心技能。
